## 手順

1. **対象プランファイルを特定する**
   - 引数でパスが指定されている場合 → そのパスを採用する（相対パスはcwd基準）
   - 引数が無い場合 → `python3 scripts/find_latest_plan.py` を実行し、現在のプロジェクトの最近のセッションログから直近に扱われたプランファイルのパスを取得する
     - パスが出力された場合 → そのプランを実装してよいかユーザーに確認してから進める
     - 何も出力されなかった場合（非ゼロ終了）→ ユーザーにプランファイルのパスを尋ねる

2. **プランファイルを読む**

   特定したパスを Read でそのまま読み込む。

3. **プラン通りに実装する**

   プランに書かれた設計・手順に沿ってコードを実装する。プランに検証手順（テスト実行・動作確認など）が含まれる場合はそれも実施する。

4. **完了報告を行う**

   コミット・push・PR作成などのGit操作は、プランに明記されている場合のみプラン通りに実施する。プランに記載がなければ行わない。実装が完了したら、変更したファイルと実施内容を簡潔に報告する。
