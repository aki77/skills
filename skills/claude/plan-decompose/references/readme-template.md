# README.md（フェーズ索引）テンプレート

`README.md` は **実装計画（フェーズ索引と各フェーズの詳細への入口）**。「どの順でどう作るか」を定義し、仕様の再定義はしない。

```markdown
# <プロジェクト/機能名> 実装フェーズ索引

このディレクトリには <プロジェクト/機能名> の実装計画を置く。役割分担は次のとおり:

- [overview.md](overview.md) — **設計仕様の single source of truth（SSOT）**。「何を作るか」をフェーズに依存しない形で定義する。
- 本 README と各 `phase-N-*.md` — **実装計画（フェーズ索引と各フェーズの詳細）**。「どの順でどう作るか」を定義する。仕様の再定義はせず、overview.md の該当節（§番号）を参照する。

各フェーズドキュメントを読む前に、必ず [overview.md](overview.md) を先に読むこと。

---

## フェーズ一覧

| # | タイトル | 要約 | 依存 |
|---|---|---|---|
| [P1](phase-1-xxx.md) | <タイトル> | <1行要約> | なし |
| [P2](phase-2-xxx.md) | <タイトル> | <1行要約> | P1 |
| ... | ... | ... | ... |

---

## 依存グラフ

\`\`\`mermaid
graph TD
    P1["P1 <タイトル>"] --> P2["P2 <タイトル>"]
    P2 --> P3["P3 <タイトル>"]

- **クリティカルパス**: 最長の依存チェーンを明記する。
- **並行可能**: 同時に進められるフェーズの組み合わせを明記する。

---

## 各フェーズドキュメントへのリンク

- [phase-1-xxx.md](phase-1-xxx.md) — P1 <タイトル>
- [phase-2-xxx.md](phase-2-xxx.md) — P2 <タイトル>
- ...

---

## 共通テンプレート（各フェーズドキュメントの節構成）

各 `phase-N-*.md` は以下の節構成に統一する（詳細は `references/phase-template.md` 参照）:

1. **目的**
2. **スコープ**
3. **前提・依存**
4. **実装方針**
5. **変更ファイル**
6. **受け入れ条件**
7. **テスト観点**
8. **リスク・決定事項**
9. **参照**

---

## overview.md「実装時に確定させる決定事項」→ フェーズのマッピング

| 決定事項 | 確定フェーズ |
|---|---|
| <決定事項1> | P2 |
| <決定事項2> | P4（枠は P3 で先出し） |
| ... | ... |
```

## 書き方のポイント

- **依存グラフとフェーズ一覧表の「依存」列は必ず一致させる**。ずれると実装順序を誤らせる。
- **並行可能なフェーズは明示する**。依存グラフだけでは「並行できる/できない」が読み取りにくいので、文章でも補足する。
- **決定事項マッピング表は overview.md の項目と1対1対応させる**。overview.md側に決定事項を追加したらこの表も更新する（漏れがないかは分割完了後にチェックする）。
